Single-Switch OC Faults Produce Semicircular Trajectories

Research gap
Example: Sa1 fault
The Sa1 open-circuit fault changes the phase-A conduction path and produces a semicircular current-vector trajectory.
General single-switch OC fault trajectories
Proposal Fig. 1.2. Healthy mode forms a circle, while each single-switch OC fault produces a semicircle with a distinct angular orientation.
Key observation
  • Under healthy balanced operation, the current vector forms a complete circle.
  • A single-switch OC fault removes one phase-current conduction interval.
  • The remaining trajectory becomes a semicircle.
  • The semicircle orientation provides a geometric signature for fault detection and fault locating.

Unbalanced Loading Distorts Current-Vector Trajectory Signatures

Research gap
Trajectory matrices under different operational modes and loading conditions
Proposal Fig. 2.5. Balanced and phase-C unbalanced loading generate different trajectory-matrix patterns for the same inverter operational mode labels.
Key observation
  • Under balanced loading, trajectory matrices show clearer mode-dependent patterns.
  • Under extreme unbalance, the trajectory can shrink and become more sensitive to noise.
  • The diagnosis method must be robust to both fault mode and loading condition.
Unbalance metric
PCUR = |Iavg − Imin| / Iavg × 100%
PCUR quantifies how strongly the smallest phase-current amplitude deviates from the average phase-current amplitude.

Mean-Vector Features Become Fragile Under Unbalanced Loading

Research gap
Mean current vector location in the a–b–c domain
Proposal Fig. 2.6. Under unbalanced loading, mean-vector locations move closer to threshold boundaries and become vulnerable to noise-induced crossing.
Mean-vector feature
μm = 1/N · Σk=1N im(k),  m ∈ {a,b,c}
  • Mean-vector methods use average current displacement as a handcrafted diagnostic feature.
  • Fault locating often depends on predefined thresholds.
  • When one phase current is very small, noise can occupy a larger fraction of the feature value.
  • Threshold crossing becomes unstable under extremely unbalanced loading.
This motivates using trajectory-matrix learning rather than relying only on manually selected mean-vector features.

From Current Vector Trajectory to Trajectory Matrix

Work 1
Encoding current-vector trajectories into a binary matrix
Proposal Fig. 2.1. Current-vector samples in the α–β domain are projected onto a discretized binary trajectory matrix.
Encoding rule
XD×D(m,n) = 1, if m = ⌊ Iα(ti) · D/4 + D/2 ⌋ n = ⌊ Iβ(ti) · D/4 + D/2 ⌋ otherwise, XD×D(m,n) = 0
  • The Clarke transform maps three-phase currents into the α–β plane.
  • Each sampled current vector activates one pixel in the trajectory matrix.
  • The bias term D/2 places the trajectory at the matrix center.
  • The scaling term D/4 keeps trajectories bounded under amplitude changes.
This converts current waveforms into an image-like spatial representation suitable for CNN feature learning.

Proposed CNN-Based Diagnosis Framework

Work 1
Proposed diagnosis methodology
Proposal Fig. 2.9. Three-phase current data are transformed into trajectory matrices and classified by CNN-based fault detection and locating modules.
Pipeline overview
  • Input: three-phase current measurements from the inverter.
  • Transformation: Clarke transform maps current samples into the α–β domain.
  • Representation: current-vector samples generate a binary trajectory matrix.
  • Learning: CNN extracts spatial trajectory features through convolution and pooling.
  • Output: fault detection and locating produce the switch-level operational-mode label.
This framework replaces handcrafted trajectory rules with learned spatial features for robust 22-class OC fault diagnosis.

Online Diagnosis: Fault Detection Followed by Fault Locating

Work 1
Detection and locating during mode transient
Proposal Fig. 2.10. Fault detection is triggered first; fault locating is activated after one fundamental cycle.
Online diagnosis logic
  • Fault detection CNN identifies whether the inverter has entered a faulty mode.
  • A trigger signal starts the fault-locating stage.
  • A one-fundamental-cycle delay ensures the locating trajectory matrix contains sufficient post-fault information.
  • Fault locating CNN outputs the six-switch operational-mode label.
Step 1: detect non-circular trajectory distortion.
Step 2: wait for one complete post-fault cycle.
Step 3: locate the faulty switch-level mode.
This separates fast fault occurrence detection from switch-level fault locating.

Why Trajectory-Matrix Learning Preserves Fault Information

Work 1
Feature extraction mechanisms
Proposal Fig. 2.11. Conventional FFT + BN separates feature extraction and classification, while the proposed method learns features and classification jointly.
Key idea
  • Traditional feature extraction compresses current signals before classification.
  • Handcrafted features may lose discriminative information under extreme unbalance.
  • The proposed method preserves the spatial trajectory pattern before CNN learning.
  • CNN training jointly optimizes feature extraction and classification for the diagnosis target.
Conventional:
manual feature extraction → fixed classifier
Proposed:
trajectory matrix → learned spatial features
This is the key reason the CNN-based approach is more robust when trajectory distortion is caused by both OC faults and loading imbalance.

Energy barrier

Where CNN Energy Is Wasted in Sparse Trajectory Matrices

Transition to Work 2
Redundant convolution on zero-valued trajectory regions
Proposal Fig. 2.12. Dense CNN convolution still processes zero regions, while spike-based computation can avoid events where no spikes are generated.
Why this matters here
  • Each trajectory matrix stores only a small set of active current-vector pixels.
  • Standard CNN convolution applies kernels over the entire matrix, regardless of sparsity.
  • Many MAC operations occur on zero-valued or non-informative regions.
  • This creates unnecessary arithmetic and memory movement for embedded monitoring.
